## 执行方式

1. 每批先输出候选问题列表。
2. 只修改可明确判定错误、过强或重复的关系。
3. 每批修改后运行：
   - JSON 解析检查
   - 关系完整性检查
   - 规避分类检查
4. 全部完成后运行：
   - `node scripts/validate/references.mjs`
   - `npm run build`
